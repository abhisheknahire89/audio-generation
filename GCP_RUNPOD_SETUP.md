# Deploying OPD Audio Generator to Cloud GPU (GCP / RunPod)

Running 100 scripts on a Mac will take over **4 days**. By offloading this to an NVIDIA L4 or A10G Cloud GPU instance, you can finish all 100 scripts in roughly **3 to 4 hours**, for a total cost of **~$3.00 to $5.00**.

Since your associate already uses the `trycloudflare.com` tunnel link, the transition is seamless—they just connect to the new link generated by the cloud server.

---

## Option 1: RunPod (Recommended - Cheapest & Easiest)

RunPod specializes in cheap, hourly, pre-configured GPU instances.

### 1. Rent the GPU
1. Create an account on [RunPod.io](https://runpod.io/).
2. Add ~$10 of billing credits.
3. Go to **Pods** -> **Deploy Secure Cloud**.
4. Select an **1x RTX 4090**, **1x L4**, or **1x A40/A10G** (whichever is available and cheapest, usually ~$0.50 - $0.80/hr).
5. For the Template, select **RunPod PyTorch** (this has Python and NVIDIA drivers pre-installed).
6. Click **Deploy**.

### 2. Start the Server
1. Click **Connect** on your Pod and open the **Web Terminal**.
2. Run the following commands to download your code and install everything:
   ```bash
   # Clone your repository
   git clone https://github.com/abhisheknahire89/audio-generation.git
   cd audio-generation
   
   # Run the standard setup
   chmod +x start.sh
   ./start.sh
   ```
3. The setup script will automatically install `f5-tts`, `parler-tts`, and the necessary dependencies.
4. Once the server starts, it will output the `trycloudflare.com` tunnel URL. Give this to your associate!

### 3. Queue the 100 Scripts
1. Have your associate upload the massive batch of scripts.
2. The server will churn through them overnight.
3. When the queue is finished, download the ZIP file.
4. **CRITICAL:** Go back to RunPod and click **Stop** and then **Terminate** the Pod so you are no longer billed!

## Option 2: Google Cloud Platform (GCP)

If you already use GCP for enterprise infrastructure, you can spin up a Compute Engine instance.

### 1. Rent the GPU
1. Go to **Compute Engine** -> **VM Instances** -> **Create Instance**.
2. Under Machine Configuration, select **GPUs**.
3. Choose **NVIDIA L4** (1 GPU).
4. For the Boot Disk, click Change, go to the **Deep Learning on Linux** tab, and select **Deep Learning VM with CUDA 12.x**.
5. Create the VM.

### 2. Start the Server
1. SSH into the VM instance directly from the GCP console.
2. Clone your repository:
   ```bash
   git clone https://github.com/abhisheknahire89/audio-generation.git
   cd audio-generation
   ```
3. Because the Deep Learning VM comes with `conda`, you might need to create an environment:
   ```bash
   conda create -n opd python=3.11 -y
   conda activate opd
   ```
4. Run the starter script:
   ```bash
   chmod +x start.sh
   ./start.sh
   ```

5. Once you are finished generating, **Stop the VM** from the GCP console. You do not pay for the GPU when the instance is stopped.
